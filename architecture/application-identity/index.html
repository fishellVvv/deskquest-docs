    Identidad de aplicacion¶

    Estado¶

    • aceptada
    • fecha de decision: 2026-03-24

    Objetivo¶

    Fijar una fuente de verdad unica para la identidad publica y tecnica del proyecto antes de abrir una beta compartible mas seria.

    La decision ya esta aceptada y el rename operativo a DeskQuest ya ha quedado cerrado:

    • runtime, persistencia y app id ya usan la identidad nueva
    • repo remoto y carpeta principal ya usan deskquest
    • la primera build empaquetada ya sale como DeskQuest
    • el siguiente paso del proyecto es activar el versionado publico profesional

    Identidad aprobada¶

    Capa Valor aprobado
    nombre comercial visible DeskQuest
    repositorio deskquest
    carpeta principal deskquest
    ejecutable interno DeskQuest
    save/config namespace deskquest
    app id com.fishellvvv.deskquest

    Motivacion¶

    La identidad provisional test_game ya no es suficiente para una fase beta con builds empaquetadas, documentacion publica y futura venta.

    Fijar esta decision ahora reduce riesgo en:

    • rutas de instalacion y guardado
    • branding de builds y releases
    • nombre del proceso y del ejecutable
    • documentacion publica
    • futura publicacion del portal y del repo

    Nombres descartados o provisionales¶

    • test_game nombre provisional interno del proyecto
    • Realmfold
    • M-idle Age

    Criterio de separacion¶

    Nombre comercial¶

    DeskQuest

    Es el nombre que debe ver el jugador en documentacion publica, builds, storefronts y comunicacion del producto.

    Slug tecnico¶

    deskquest

    Debe gobernar repo, carpeta principal, namespaces tecnicos y claves persistentes nuevas cuando se haga la migracion.

    App id¶

    com.fishellvvv.deskquest

    Se acepta como identificador tecnico estable para Tauri y para las carpetas de perfil asociadas a la app.

    Nota:

    • el prefijo fishellvvv se acepta por ahora como suficientemente estable
    • no hace falta fijar todavia un nombre formal de estudio para cerrar esta capa

    Alcance de la decision¶

    Esta decision significa que, a partir de ahora:

    • DeskQuest es el nombre correcto para hablar del producto
    • deskquest es el slug tecnico aprobado
    • com.fishellvvv.deskquest es el app id activo

    Pero no significa todavia que el cierre del rename ya haya terminado del todo.

    Estado actual de la migracion¶

    La migracion tecnica desde test_game ya esta cerrada a efectos operativos.

    Quedan solo dos reglas vivas:

    • mantener compatibilidad legacy donde aun existan claves o perfiles viejos
    • activar justo despues el versionado publico profesional

    Riesgos que justifican hacerlo como bloque aparte¶

    • cambiar el app id mueve rutas de perfil del usuario
    • cambiar namespaces de guardado puede romper persistencia existente
    • cambiar repo y branding a medias deja trazabilidad sucia
    • una build beta compartida con identidades mezcladas complica soporte y QA

    Regla operativa a partir de ahora¶

    Con el rename ya cerrado:

    • el codigo puede seguir conteniendo test_game solo en rutas y claves legacy de compatibilidad o en el historico documental
    • la documentacion debe referirse al producto como DeskQuest
    • cualquier cambio de identidad tecnica debe seguir esta pagina como fuente de verdad
