    Instalacion y datos de usuario¶

    Esta pagina resume donde vive la app empaquetada, donde guarda sus datos y que conviene copiar si quieres hacer backup o mover una partida.

    Nota sobre el estado actual¶

    La migracion tecnica principal hacia DeskQuest ya esta ejecutada.

    Ahora mismo:

    • nombre visible del producto: DeskQuest
    • app id activo: com.fishellvvv.deskquest
    • namespace web activo: deskquest.*
    • perfiles heredados de io.github.fishellvvv.testgame se migran automaticamente al arrancar una build nueva si el perfil nuevo aun no existe

    Los nombres antiguos siguen siendo relevantes solo como rutas legacy de origen para soporte y compatibilidad.

    1. Que se instala realmente¶

    La app empaquetada instala el binario de DeskQuest, pero la ruta exacta de instalacion puede variar segun el tipo de bundle y la configuracion del equipo.

    Lo importante para soporte diario no suele ser la carpeta del instalador, sino estas dos rutas:

    • perfil local en AppData\Local\com.fishellvvv.deskquest
    • perfil roaming en AppData\Roaming\com.fishellvvv.deskquest

    Si necesitas documentar una build concreta para testers, revalida la ruta real de instalacion con ese instalador en particular.

    2. Donde vive el perfil local de la app¶

    La app guarda su perfil local en:

    C:\Users\<usuario>\AppData\Local\com.fishellvvv.deskquest

    Contenido habitual:

    • EBWebView
    • logs

    Notas:

    • EBWebView es el perfil del WebView embebido.
    • logs contiene los logs nativos o auxiliares que la app vaya generando.

    Ruta legacy de origen¶

    Si vienes de builds anteriores, el perfil antiguo podia vivir en:

    C:\Users\<usuario>\AppData\Local\io.github.fishellvvv.testgame

    La app intenta migrarlo automaticamente al primer arranque de la version nueva.

    3. Donde esta la partida guardada¶

    La partida no se guarda como un .json plano facil de editar. Ahora mismo vive dentro del almacenamiento local del WebView:

    C:\Users\<usuario>\AppData\Local\com.fishellvvv.deskquest\EBWebView\Default\Local Storage\leveldb

    Importante:

    • aqui viven progreso y configuracion persistida del mundo
    • el formato real es LevelDB, no un savefile legible a mano
    • no se recomienda editar esta carpeta manualmente salvo para soporte o backup

    Si una build vieja ya tenia partida guardada bajo el perfil anterior, el origen legacy equivalente seria:

    C:\Users\<usuario>\AppData\Local\io.github.fishellvvv.testgame\EBWebView\Default\Local Storage\leveldb

    4. Donde estan los paquetes de dimensiones del usuario¶

    Ademas del perfil interno de la app, DeskQuest siembra una carpeta publica para paquetes .dimension en:

    C:\Users\<usuario>\Documents\DeskQuest\dimensions

    Ahí es donde la app deja:

    • lunar-base.dimension
    • LEEME - dimensiones.txt
    • y cualquier otro .dimension que el usuario quiera guardar para importarlo

    Notas:

    • esta carpeta es la via pensada para compartir e intercambiar dimensiones
    • la app no activa Base lunar por si sola: hay que importarla desde Configuracion -> Dimensiones -> Anadir dimension...
    • si Documents no estuviera disponible, la app puede usar un fallback interno bajo AppData

    5. Donde estan otros datos del usuario¶

    Ademas del perfil local, la app puede usar una carpeta de roaming en:

    C:\Users\<usuario>\AppData\Roaming\com.fishellvvv.deskquest

    Contenido habitual o esperado:

    • installed-dimensions
    • dimension-archive-cache
    • telemetry
    • caches auxiliares de dimensiones ya importadas

    Ruta legacy equivalente:

    C:\Users\<usuario>\AppData\Roaming\io.github.fishellvvv.testgame

    5.1 Como limpiar el legado una vez validada la beta¶

    Cuando DeskQuest ya:

    • arranca bien
    • conserva progreso
    • y tiene backup externo hecho

    se pueden retirar los restos legacy de test_game.

    Rutas a eliminar:

    C:\Users\<usuario>\AppData\Local\io.github.fishellvvv.testgame
C:\Users\<usuario>\AppData\Roaming\io.github.fishellvvv.testgame
C:\Users\<usuario>\AppData\Local\test_game

    Reglas:

    1. cerrar la app por completo
    2. confirmar que existe backup externo en DeskQuest-backups
    3. borrar solo esas rutas legacy
    4. no tocar:
    C:\Users\<usuario>\AppData\Local\DeskQuest
C:\Users\<usuario>\AppData\Local\com.fishellvvv.deskquest
C:\Users\<usuario>\AppData\Roaming\com.fishellvvv.deskquest

    Una vez ejecutada la limpieza, los nombres test_game y io.github.fishellvvv.testgame deberian sobrevivir solo en:

    • backups historicos
    • documentacion de migracion
    • referencias legacy de soporte

    6. Que copiar para hacer backup¶

    Solo progreso y configuracion¶

    Copia esta carpeta:

    C:\Users\<usuario>\AppData\Local\com.fishellvvv.deskquest

    Perfil mas completo¶

    Si ademas quieres conservar dimensiones instaladas y datos auxiliares, copia tambien:

    C:\Users\<usuario>\AppData\Roaming\com.fishellvvv.deskquest

    7. Como restaurar un backup¶

    Flujo recomendado:

    1. cerrar la app por completo
    2. restaurar las carpetas de Local y, si hace falta, Roaming
    3. volver a abrir la app

    Si la app estaba abierta mientras se restauran datos, es facil que el perfil se reescriba al cerrar.

    Ruta recomendada para backups externos¶

    Para no mezclar seguridad y runtime activo, el lugar recomendado para backups manuales fuera de AppData es:

    C:\Users\<usuario>\OneDrive\Documentos\DeskQuest-backups

    Buenas practicas:

    • usar subcarpetas con timestamp
    • separar current y legacy mientras aun exista el perfil antiguo
    • no restaurar ni borrar perfiles con la app abierta

    8. Que pasa si reseteas o borras el perfil¶

    Si renombras o eliminas:

    C:\Users\<usuario>\AppData\Local\com.fishellvvv.deskquest\EBWebView

    la app creara un perfil nuevo al arrancar.

    Eso suele resolver perfiles rotos del WebView, pero tambien implica:

    • perder progreso guardado
    • perder configuracion persistida del mundo
    • regenerar el perfil del navegador embebido

    9. Relacion con la build y con testers¶

    La guia de testers no se empaqueta dentro del .exe.

    Ahora mismo:

    • la app visible se presenta como DeskQuest
    • el perfil activo vive bajo com.fishellvvv.deskquest
    • el perfil legacy io.github.fishellvvv.testgame solo debe tratarse ya como origen de migracion o ruta historica
    • antes de limpiar el legado, conviene dejar una copia externa bajo DeskQuest-backups
    • despues de la limpieza, esas rutas legacy pueden no existir ya en la maquina viva y quedar solo en el backup externo
    • la documentacion sigue yendo aparte
    • si compartes una build con testers, conviene darles tambien el enlace o copia de la Guia corta para testers
    • el portal publico activo vive en https://fishellvvv.github.io/deskquest-docs/
    • el canal publico de feedback beta vive en https://github.com/fishellVvv/deskquest-docs/issues
