    Presets de desarrollo¶

    Esta pagina explica para que sirven los presets de mundo y como usarlos sin mezclarlos con las tablas de balance.

    1. Que problema resuelven¶

    Durante el desarrollo hay muchas veces en las que no interesa recorrer toda la progresion del juego solo para probar una pantalla, una construccion o una produccion concreta.

    Los presets existen para eso:

    • arrancar el juego directamente en un estado concreto del reino
    • probar onboarding, construcciones o produccion sin rehacer todo a mano
    • editar snapshots JSON rapidamente para testing

    2. Que cambian y que no cambian¶

    Los presets cambian el WorldState inicial en modo desarrollo.

    Pueden fijar, por ejemplo:

    • fase del mundo
    • oro
    • nombre del asentamiento
    • poblacion total y asignada
    • recursos acumulados
    • edificios completados

    No cambian:

    • las tablas .csv de balance
    • las dimensiones instaladas
    • la configuracion general de la app
    • la logica de juego

    3. Balance vs presets¶

    La regla importante es esta:

    • balance/ en .csv = numeros estaticos y calibrables
    • dev-presets/world/ en .json = snapshots de progreso para testing

    Ejemplos de datos que van en CSV:

    • coste de un edificio
    • tiempo de construccion
    • tasa base de produccion
    • multiplicadores

    Ejemplos de datos que van en presets JSON:

    • cuanta madera tiene ahora el reino
    • si el puesto maderero ya esta construido
    • cuantos trabajadores tiene asignados el puesto maderero
    • como se llama el asentamiento

    4. Donde viven¶

    Los archivos estan en:

    • selector activo: dev-presets/active-world-preset.txt
    • presets editables: dev-presets/world/

    5. Comandos¶

    Ver la lista disponible¶

    npm run dev:preset:list

    Muestra:

    • los presets disponibles
    • cual esta activo ahora mismo

    Activar un preset¶

    npm run dev:preset -- foundation-ready

    Esto no modifica la app ya abierta al instante.

    La idea es:

    1. activar preset
    2. reiniciar la app
    3. arrancar con ese estado

    Volver al flujo normal¶

    npm run dev:preset:none

    Desactiva el preset forzado y devuelve la app al progreso normal guardado.

    6. Flujo recomendado¶

    Ejemplo para probar el puesto maderero:

    1. activar preset
    npm run dev:preset -- lumber-producing
    1. cerrar la app si estaba abierta
    2. volver a arrancar
    npm run dev
    1. comprobar el estado
    2. cuando termines, desactivar preset
    npm run dev:preset:none
    1. reiniciar la app otra vez

    6.1 Flujo rapido para volver a Dirigir tala¶

    Si quieres repetir el evento de tala sin rehacer toda la progresion temprana, usa este alias:

    npm run dev:reset-logging

    Eso activa el preset logging-ready.

    Luego:

    1. cierra la app si estaba abierta
    2. vuelve a arrancarla
    3. entra en Puesto maderero -> Dirigir tala

    Cuando termines tus pruebas, vuelve al flujo normal con:

    npm run dev:preset:none

    7. Presets actuales¶

    journey-fresh¶

    Arranca en el viaje inicial.

    Util para:

    • probar la escena de carreta
    • repetir onboarding desde cero

    foundation-ready¶

    Arranca con oro suficiente para fundar enseguida.

    Util para:

    • probar la fundacion
    • revisar tiempos de construccion iniciales

    stronghold-named¶

    Arranca con el asentamiento ya fundado y nombrado.

    Util para:

    • probar el patio sin rehacer fundacion
    • revisar navegacion del reino temprano

    lumber-producing¶

    Arranca con el puesto maderero construido y produciendo.

    Util para:

    • probar produccion de madera
    • revisar asignacion de trabajadores
    • validar persistencia de recursos

    logging-ready¶

    Arranca con el reino justo en el punto de entrada a Dirigir tala.

    Util para:

    • repetir el nivel 1 de tala sin rehacer onboarding ni mejoras previas
    • probar setup de zona/nivel/cuadrillas, pausa, reintento y economia de expedicion (obtenido / acumulado)
    • validar cambios en el pergamino o en la escena jugable del maderero

    quarry-producing¶

    Arranca con cantera, viviendas y produccion de piedra ya activas.

    Util para:

    • probar piedra sin rehacer todo el early game
    • revisar competencia de trabajadores entre maderero y cantera
    • validar aforo y crecimiento de poblacion con el reino ya mas avanzado

    housing-upgrade-ready¶

    Arranca con Viviendas en nivel 1 y recursos suficientes para lanzar su primera mejora.

    Util para:

    • probar el sistema generico de niveles sin rehacer todo el patio
    • validar el nuevo aforo tras la mejora
    • comprobar que poblacion y Viviendas siguen coherentes al reiniciar

    8. Como crear o editar uno¶

    Puedes:

    1. duplicar uno de los JSON existentes
    2. cambiar su contenido
    3. guardarlo con otro nombre
    4. activarlo por terminal

    Ejemplo:

    • copiar dev-presets/world/foundation-ready.json
    • renombrarlo a quarry-ready.json
    • ajustar oro, edificios y recursos
    • ejecutar:
    npm run dev:preset -- quarry-ready

    9. Regla practica¶

    Si quieres tocar:

    • costes, tiempos, tasas o multiplicadores -> usa CSV
    • progreso del reino para pruebas rapidas -> usa preset JSON

    Esa separacion mantiene el proyecto limpio y hace el testing mucho mas rapido.
