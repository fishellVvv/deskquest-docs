    Polish previo a compartir la beta¶

    Esta pagina agrupa los remates no funcionales que siguen pendientes antes de compartir DeskQuest con colegas cercanos.

    No abre un nuevo frente de desarrollo grande. Su funcion es cerrar una ronda corta de:

    • usabilidad
    • estetica
    • facilidad de acceso

    para que la siguiente beta compartible resulte mas facil de instalar, entender y probar.

    Estado¶

    • estado general: cerrado
    • fase: polish previo a compartir beta
    • relacion con la operativa:
    • este checklist materializa el paso 16 ya cerrado de la Operativa de cierre de beta
    • con los bloques principales ya cerrados y la release publica ya viva, el siguiente gesto ya no es mas polish estructural, sino compartir la beta y recoger feedback antes de decidir el siguiente slice funcional

    Criterio de cierre¶

    Esta ronda se podra dar por cerrada cuando:

    • la beta se descargue y entienda con menos friccion
    • la capa de configuracion resulte mas clara
    • el game no exponga controles impropios o confusos
    • los idle principales esten ya en una presentacion aceptable
    • la dimension base lunar quede revisada dentro del paquete beta

    Bloques de trabajo¶

    1. Acceso y distribucion¶

    Impacta directamente en que un tester pueda entrar sin perderse.

    Pendientes¶

    • [x] hacer que la documentacion publica incluya el link de descarga del ejecutable
    • [x] establecer un enlace a la documentacion en el juego
    • [x] establecer una primera pagina de la documentacion que de un breve resumen y pocos enlaces para evitar abrumar en exceso
    • [x] hacer que la beta deje lunar-base.dimension y un LEEME inicial en la carpeta publica de dimensiones

    Complejidad estimada¶

    • baja: link de descarga en la documentacion publica
    • baja/media: nueva portada publica mas simple para docs
    • media: enlace a documentacion dentro del juego
    • media: siembra publica de base lunar y su carpeta de paquetes

    2. Configuracion y usabilidad¶

    Es el bloque con mas retorno inmediato antes de compartir la beta.

    Pendientes¶

    • [x] activar algunos controles de configuracion utiles
    • [x] mostrar muteados los controles de configuracion que no funcionen
    • [x] permitir el plegado y despliegue de las categorias de configuracion
    • [x] hacer que el boton de settings te permita salir de settings

    Resultado actual¶

    • General: Idioma y Comportamiento al cerrar activos; Iniciar con Windows y Frecuencia de guardado siguen muteados
    • Escritorio: Monitor del idle, Posicion del idle, Tamano de la base, Siempre visible y Transparencia del idle activos; Iniciar con Windows sigue muteado
    • Audio: bloque rehecho visualmente como General, Efectos, Ambiente y Musica, pero sigue muteado de forma honesta
    • Actividad: Contar actividad fuera de la app y Animaciones activas activos; Frecuencia de actividad, Sutileza visual y Notificaciones siguen muteados
    • Interfaz: Atajos de teclado activo; Escala del dashboard, Estilo del panel, Formato numerico y Modo capturas / stream siguen muteados
    • Accesibilidad: Reducir movimiento activo; Tamano de texto y Alto contraste siguen muteados
    • Avanzado: Confirmar acciones delicadas y Modo desarrollador siguen muteados; la Zona de peligro ya es funcional

    Complejidad estimada¶

    • media: boton de settings como entrada y salida
    • media: mostrar muteados los controles no operativos
    • media: plegado y despliegue de categorias
    • media/alta: decidir y activar los controles utiles reales para esta beta

    3. UX del game y del pergamino¶

    Son remates pequenos, pero muy visibles para cualquiera que pruebe Dirigir tala.

    Pendientes¶

    • [x] quitar el boton de settings y el de cerrar del pergamino cuando estas en un game
    • [x] revisar tooltips y menu del click derecho

    Resultado actual¶

    • mientras haya mision abierta, el pergamino ya no muestra toolbar global de settings ni cerrar
    • el click derecho ya usa un menu contextual propio en el dashboard
    • durante un game activo, ese menu se reduce a Cerrar aplicacion
    • en idle y en la puerta/hotspot no se abre menu contextual
    • el patio y Dirigir tala ya han pasado por una segunda ronda de tooltips para explicar mejor acciones, estado y contexto sin limitarse a repetir etiquetas

    Complejidad estimada¶

    • baja: quitar botones impropios del pergamino durante un game
    • media: revisar tooltips y menu contextual

    4. Contenido y presentacion visual del idle¶

    Aqui entran cambios visibles de primera impresion, pero no tan criticos como el acceso o la configuracion.

    Pendientes¶

    • [x] cambiar idle de carreta
    • [x] cambiar idle de construccion
    • [x] cambiar idle de asentamiento

    Resultado actual¶

    • journey, founding y stronghold ya usan PNG propios de 320x160
    • el idle ya no muestra atmosferas, pulsos ni glow superpuestos; se ve limpio como raster + puerta
    • las tres fases comparten ya una hitbox comun de puerta 40x40, centrada abajo
    • en founding, entrar en Configuracion ya renderiza correctamente el cuerpo de ajustes aunque el nombrado del asentamiento siga pendiente

    Complejidad estimada¶

    • media si ya existen variantes o direction clara
    • media/alta si aun hay que decidir o producir esas versiones visuales

    5. Dimensiones y contenido base¶

    Bloque de revision de calidad del paquete inicial.

    Pendientes¶

    • [x] revisar la dimension de base lunar

    Resultado actual¶

    • Base lunar ya no aparece dentro del juego como instalacion oficial especial
    • el primer arranque siembra lunar-base.dimension y un LEEME en la carpeta publica de paquetes
    • la app solo la muestra en runtime despues de importarla con Anadir dimension
    • la seccion Dimensiones ya queda simplificada a Instaladas, activacion por boton y acciones directas de Anadir dimension / Quitar dimension
    • el enlace embebido abre ya la guia publica de modding directamente en la documentacion de dimensiones
    • el .dimension oficial editable, el embebido y la copia publica sembrada vuelven a estar alineados con autor DeskQuest

    Complejidad estimada¶

    • media, salvo que aparezcan problemas de empaquetado o compatibilidad

    Orden recomendado¶

    Si hay que empezar ya, este es el orden sugerido:

    1. acceso y distribucion
    2. configuracion y usabilidad
    3. UX del game y del pergamino
    4. dimension base lunar
    5. cambios visuales del idle

    Primera sugerencia de ejecucion¶

    Si queremos maximizar valor rapido antes de compartir la beta, yo empezaria por:

    1. link de descarga en la documentacion publica
    2. portada publica mas simple para la documentacion
    3. comprobar que el instalador incluye base lunar
    4. enlace a documentacion dentro del juego

    Despues pasaria a la capa de configuracion.

    Estado de cierre actual¶

    La ronda principal de polish previo ya queda completada en:

    • acceso y distribucion
    • configuracion y usabilidad
    • UX del game y del pergamino
    • dimensiones y contenido base
    • presentacion visual del idle

    El siguiente paso ya no es seguir retocando esta lista, sino decidir si la beta puede compartirse tal como esta, si conviene anadir pequenos atajos secretos para devs/testers o si ya toca declarar la ronda congelada.

    Relacion con GitHub Project¶

    Este checklist deberia traducirse en tarjetas concretas dentro de DeskQuest Beta antes de empezar a ejecutarlo, para no perder el seguimiento operativo.
